# Delaware Hybrid Funding Formula Calculator

A single-file HTML calculator that models Delaware's K-12 school funding formula — Division I units, school and Central Office positions, and the PEFC Hybrid Funding Model (Opportunity + Operational funding) — for every district, charter, and school in the state.

**Not an official DDOE tool.** This is an independent working model built to make the funding formula's business rules visible, traceable to source, and testable — not a substitute for DDOE's own calculations.

## Governing source of truth

This calculator is built to be faithful to **`Delaware School Funding Formula Technical Reference.xlsx`**, specifically its **Rule Register** sheet. That spreadsheet is the authoritative record: every formula, rate, threshold, status, and legal citation in `index.html` should trace back to a Rule ID in that Rule Register. If the calculator and the Rule Register ever disagree, the Rule Register is what's correct — the calculator needs to be fixed, not the other way around.

The Rule Register's "Data Source System" column, in turn, cites `Hybrid Funding Formula Calculator for 25-26 w Charter version 05152026.xlsm` as the underlying evidence for each rule — that's where the real enrollment, position counts, and salary rates come from, and how formula behavior gets confirmed. So the relationship is: **Rule Register = what this calculator should do and why; reference workbook = the real data and evidence that formula is built and checked against.**

One more layer worth being precise about: the Hybrid Funding Model itself — Division I units, Central Office positions, Opportunity/Operational weighted funding — is **PEFC's** (the Delaware Public Education Funding Commission's) policy design. The reference workbook is where PEFC's model is recorded and computed in spreadsheet form, not an independent source of authority. So wherever this project says a rule "matches the reference workbook," the underlying decision is PEFC's; the workbook is just the artifact it's recorded in.

## Live file

`index.html` — open it directly in any browser. No build step, no dependencies, no server required. All data is embedded in the file itself.

## What's in it

The calculator has four tabs:

**Business Rule Register.** A condensed, in-browser mirror of the Rule Register sheet in the Technical Reference workbook — every funding component (Division I units, Principal, Central Office positions, Opportunity/Operational weights, etc.) listed with its formula, FY25-26 rate, and a status badge: Verified (formula and value both confirmed), Business rule confirmed (matches the reference, legal citation still pending), Open discrepancy, or Verified (value) / Open (formula) — used where the real position count is trustworthy but the generating rule itself isn't independently verified. The full Rule Register carries additional columns (legal citation URL, data snapshot date, business/legal approver, boundary test cases) that don't fit in the browser view.

**Locked Model.** Real FY25-26 data for all 259 Delaware schools and 43 independent LEAs (19 districts + 24 charters), plus a Statewide total. Pick an LEA and optionally a school, and it shows exactly what the current formula produces — enrollment, positions, and dollars — using the real numbers, read-only. Base Funding rates (Division I, Principal, Assistant Principal, Admin Support Professionals, Instructional Supports) are fixed FY25-26 values from the reference workbook — that's PEFC's decision, not something this tab lets you adjust. (Central Office position rates in this tab were already fixed.) This tab answers "what does the current formula actually produce right now" — if you want to test a different rate, use the What-If Model instead.

**What-If Model.** Type in hypothetical enrollment (K-3, 4-12, Basic, Intense, Complex, Low-Income, MLL, Vocational, and an optional Total Enrollment override) for a school or LEA that doesn't exist yet, and see the positions and funding the formula would produce. Choose Single School or Whole LEA/District scope; at LEA scope, also choose District or Charter (Assistant Superintendent is District-only). Every rate is editable, including each Division I category's rate independently and the statewide Opportunity ($163M) and Operational ($279,026,800) funding pools. Every row is tagged **Verified** (confirmed reference-workbook formula), **Pattern-derived** (a step-function fitted against the real position counts in all 259 schools — no published formula exists for Principal, Assistant Principal, or Admin Support Professionals), or **Manual entry** (Food Services Supervisor's per-school "designated site" flag — genuinely not derivable from enrollment). A **Regular Pre-K** category was considered but removed (see History) — PEFC's "New Rule Updates" tab proposes it, but the only available citation, 14 Del. C. § 1703(d)(1)a, limits the preschool unit to students eligible for special education, so it doesn't support a general-education Pre-K category under current statute. All enrollment inputs are clamped to non-negative values.

**Plan Forward.** Open items: things that are correct-per-the-reference-workbook but still conflict with cited statute (Reading Cadre's charter exclusion), formulas that couldn't be verified or derived (Principal/AP/Admin Support Professionals/Food Services Supervisor), and policy questions the reference workbook itself flags as unresolved (Equalization, Local Funding Reform, tiering of LI/MLL/Vocational weights).

## Files

**`Delaware School Funding Formula Technical Reference.xlsx`** — the governing document. Its **Rule Register** sheet (23 rows) is the source of truth for every formula, threshold, cap, status, and legal citation this calculator implements; its **Legend** sheet defines each Rule Register column. This file is maintained by hand and is not read by the calculator at runtime — `index.html` is meant to be a faithful implementation of it, kept in sync manually whenever the Rule Register changes.

**`Hybrid Funding Formula Calculator for 25-26 w Charter version 05152026.xlsm`** — the evidence source cited in the Rule Register's "Data Source System" column for each rule. All real enrollment, position, and salary-rate data embedded in `index.html` was extracted from this workbook with `openpyxl` in cached-value mode (259 schools, 43 LEAs, 1 Statewide row). Nothing in the Locked Model tab is typed in by hand.

**`index.html`** — the calculator itself, described above.

## Known limitations

- **Principal, Assistant Principal, Admin Support Professionals**: no verified closed-form formula exists (even the reference workbook stores these as precomputed values, not formulas). The What-If Model's numbers for these are pattern-derived from the real dataset and explicitly labeled as such.
- **Food Services Supervisor, individual-school level**: which specific school within an LEA is the "designated" site doesn't correlate with enrollment or size — only 8 of 259 real schools have a nonzero value at the per-school level, always exactly 1. Modeled as a manual yes/no toggle in the What-If tab. (The separate LEA-level Food Services Supervisor position *does* have a confirmed formula — see below.)
- **Assistant Superintendent's District-only eligibility is PEFC's own confirmed rule but not empirically testable**: no charter LEA's combined Division I unit base reaches the 300-unit threshold in the current real data (highest is ~286), so the charter-exclusion claim has never actually been triggered in real data, even though PEFC has stated this is their intended direction. Director, which uses the same LEA-wide calculation base in the current workbook but a different stepped formula, is empirically confirmed to apply to both districts and charters — worth a direct confirmation with DDOE on whether that eligibility asymmetry is intentional.
- **Assistant Superintendent / Director / 11-Month Supervisor calculation base conflicts with PEFC's own documentation (flagged 2026-07-24, unresolved)**: PEFC's New Rule Updates tab and the statute paraphrase on the Formula Technical Reference sheet both describe the base for these three positions as "Division I state units" alone. Tested directly against every district and charter aggregate row: pure Division I units do not reproduce the real workbook's actual output, while the broader combined base this calculator uses (Division I + Principal + AP + Instructional Supports + Admin Support Professionals) matches exactly or to within ~0.004. This calculator matches what the real workbook computes, not what PEFC's plain-language description says the formula is — a genuine open conflict between stated policy and current practice that needs a direct answer from DDOE/PEFC, not something this project can resolve unilaterally.
- **11-Month Supervisor / Transportation Supervisor have intentionally different Locked and What-If behavior**: the Locked Model uses continuous division because that reproduces the current workbook's statewide reconciliation targets (107.77 and 18.84 positions). The What-If Model separately tests the statute-literal interpretation: fractional below the first threshold and floored at or above it (for example, 149 units → ~0.99 and 225 units → 1). The source `.xlsm` is not present in this project folder, so this split is documented from the extracted reference data, the workbook's reconciliation targets, and the statute text rather than a fresh inspection of the original `.xlsm`.
- **Whole-LEA scope in the What-If Model is a single combined-enrollment approximation, not a real multi-school rollup**: Principal, AP, and Admin Support Professionals are nonlinear step-function formulas, so computing them once from one combined hypothetical enrollment is not mathematically equivalent to entering several real schools and summing their positions. This is a real modeling limitation — a proper fix would require adding multi-school input support, which hasn't been built.
- **The source `.xlsm` reference workbook is not in this project folder.** The README and Rule Register cite `Hybrid Funding Formula Calculator for 25-26 w Charter version 05152026.xlsm` as the evidence source for every real data point embedded in `index.html`, but only the already-extracted reference data embedded in `index.html` is present. Current-workbook claims can be reconciled to those extracted values and the Technical Reference workbook, but cannot be freshly traced back through the original `.xlsm` formulas until that source file is restored.
- **The "Retired Rules (pre-2026-07-23)" sheet is an explicitly partial reconstruction**, with formula text truncated to 80 characters. Treat it as an audit trail of what IDs and components existed, not as usable formula reference.
- **The Rule Register's Business Owner and Legal / Policy Approver columns are blank for nearly every row.** No human is currently assigned to sign off on any of this — worth resolving before treating this calculator as more than a working model.
- **Reading Cadre**: confirmed business rule (1 per district/charter) conflicts with the cited FY2026 appropriation statute, which excludes charters. This conflict exists in the reference workbook too — it's a genuine open policy question, not a bug. See the Register tab.
- **Vocational Division I / Vocational Deduct**: pulled as precomputed values from the reference workbook rather than re-derived from the underlying CTE participation-percentage chain, to avoid introducing a new error while re-deriving a working value. In the real workbook, Vocational Deduct (Data!K) is stored as a negative number and added directly; the Locked Model mirrors that sign. The What-If Model's Vocational Deduct input is entered as a positive magnitude (it can't be negative) and is subtracted in code — fixed 2026-07-24 after a prior version added it instead of subtracting it, which inverted the deduction into an addition.
- **Dover Air Force Base**: appears in the reference workbook's data as a distinct entity but is not one of the 43 independent LEAs (its own Assistant Superintendent/Director fields are 0). Its enrollment is folded into the Statewide totals but it is not separately selectable in either calculator tab.
- **The Technical Reference workbook's "New Rule Updates" tab reflects PEFC's own confirmed direction for future rule changes.** These rules were authored by PEFC (the Public Education Funding Commission) itself, and per Zoey (2026-07-23), PEFC has stated in a meeting that this is the direction they're heading — meaningfully more authoritative than an unverified internal draft. Checked row by row against the real reference workbook: Superintendent, Administrative Assistant, Buildings & Grounds, Food Services Supervisor, Transportation, all five Division I divisors, and every Opportunity/Operational weight already match exactly. **Correction 2026-07-24:** this bullet previously also claimed Assistant Superintendent's 300-unit rule and Director already matched exactly — that overstated it; see the calculation-base conflict noted above. There is exactly one confirmed structural gap: **Regular Pre-K** is proposed by PEFC but doesn't exist as a column in the real workbook's current data. It was briefly added as a proposed What-If input, then removed on 2026-07-24 once the only available legal citation for it (§ 1703(d)(1)a) turned out to cover special-education-eligible preschoolers only, not general-education Pre-K — so it currently has no statutory support and isn't modeled anywhere in the calculator. There's also one genuinely open item New Rule Updates flags in its own preamble but doesn't resolve: whether Operational Funding's Enrollment weight should apply to total student enrollment or only regular-education enrollment. This calculator uses total enrollment (matching New Rule Updates' own stated scope note), but that population-basis question is PEFC's own open item, not something settled here.
- **Rule ID hygiene**: on 2026-07-22 the Rule Register's original 38 rows were overwritten in place with a reconciled 23-row set without archiving the originals first — a violation of the Legend's own "never renumber or reuse" governance rule. Repaired on 2026-07-23 (new rows moved to a BR2- namespace, retired rows partially reconstructed into a separate sheet), but that's a fix after the fact, not a substitute for doing it correctly the first time.

## History

This calculator went through several iterations before landing on a dataset-driven design: an early version used manually-entered enrollment and formulas sourced from Title 14 statute text and GPT-assisted notes, several of which turned out to deviate from the workbook DDOE and district staff actually use. It was rebuilt from the ground up against the real reference workbook on 2026-07-22 — see the discrepancy box on the Register tab for the specific corrections that resulted (removal of a standalone Pre-K bucket, Instructional Supports modeled as a flat 20% catch-all instead of individually-modeled positions, Assistant Superintendent applying equally to charters and districts, and 11-Month Supervisor/Transportation Supervisor using continuous division with no floor).

On 2026-07-23, the Technical Reference workbook's Rule Register was fully reconciled against those same corrections — the old statute-derived rows (School Nurse, Driver Education Teacher, the old fractional-credit Assistant Principal formula, the old 95-unit Buildings & Grounds threshold, a dynamic 44-LEA Statewide count) were retired, replaced with 23 rows matching what the calculator now actually implements, each citing the reference workbook as evidence. The Rule Register became the calculator's governing document at that point — previously the calculator and the Rule Register had drifted apart and only the calculator was being kept current.

Later on 2026-07-23, a review against the Technical Reference workbook's other tabs (New Rule Updates, Weight Register) surfaced two real mechanism bugs: Assistant Superintendent and Director had been described as computed per-school-then-summed, which turned out to be wrong when checked against every individual school row in the real data (every school's own field is 0 — the real number only exists on the LEA aggregate row, computed once from the LEA-wide combined base). The same review found that Food Services Supervisor, previously marked an open discrepancy, actually has a confirmed formula at the LEA level (1 + 1 per additional 500 units beyond the first 500) — the earlier "open discrepancy" conclusion came from checking the wrong granularity (per-school instead of per-LEA). Both fixes are reflected in the Locked Model, the What-If Model (which now also computes Assistant Superintendent, Director, and LEA-level Food Services Supervisor, previously omitted because the per-school mechanism was thought to make them impossible to model from a hypothetical enrollment), and the Rule Register.

Still on 2026-07-23, Zoey supplied the actual text of 14 Del. C. § 1703 (Unit of pupils). This let the K-3, 4-12, Basic, Intense, and Complex divisors move from "confirmed against the reference workbook" to directly statute-cited in the Rule Register. It also explained something that had only been an empirical observation before: § 1703(a) defines two categories at the identical 8.4 divisor — "Preschool Basic Special Education" and "K-12 Basic Special Education" — which is exactly why the reference workbook can merge them into one "Basic — Pre-K-12" line without losing anything. Special-education preschoolers are already fully funded under current law. What § 1703 does *not* define is any unit for general/regular-education Pre-K — confirming that PEFC's proposed Regular Pre-K rule really is a new category, not something already covered under a different name. The Base Funding rate inputs were also made directly editable in both calculator tabs' results tables (previously editable only via a separate card, and their derivation/year-stability still isn't independently confirmed for the non-statute-backed rates like salary figures).

On 2026-07-24, Zoey asked directly whether Regular Pre-K was actually already codified rather than proposed. Re-reading § 1703(d)(1)a showed it only covers "children between the ages of 3 and 5 who are identified as eligible for special education and related services" — not general-education Pre-K — so the "proposed, not current law" label was accurate but also meant the feature had no real legal footing. Zoey then asked to remove it entirely rather than keep an unsupported proposed input. The Regular Pre-K enrollment field, its position calculation, and its table row were removed from the What-If Model; stale references to it in the Register tab's discrepancy box, the Rule Register's BR2-003 note, and this README were updated to match. Each Division I category's rate (K-3, 4-12, Basic, Intense, Complex, Vocational Deduct, Vocational Division I) was also made independently editable in both tables' results rows — previously only the K-3 row was a real input and the rest silently mirrored it as static text — with edits to any one row syncing the shared rate across all of them.

Immediately after, Zoey clarified that this editability shouldn't have applied to the Locked Model at all: those Base Funding rates are real FY25-26 values from the reference workbook — PEFC's decision, not a hypothetical this calculator should let you change. The Locked Model's Base Funding table (Division I rows and Principal/AP/Admin Support Professionals/Instructional Supports) was reverted to fixed, read-only cells, matching the rest of that tab's "what does the current formula actually produce right now" design. The What-If Model's Base Funding rates remain fully editable and synced across the Division I rows, since that tab exists specifically to model future/hypothetical rate changes.

A review flagged two more issues on 2026-07-24, both checked against real data rather than accepted or dismissed on sight. First: the What-If Model's Vocational Deduct input couldn't accept a negative number (min=0) but was being *added* to Division I units, inverting what should be a deduction — confirmed against the real Data sheet, where Vocational Deduct is stored as a negative value in every LEA that has one. Fixed by subtracting the (positive) input instead of adding it, so the deduction now actually reduces the total, without requiring users to type a negative number. Second: the review pointed out that the Rule Register spreadsheet's stated Assistant Superintendent formula uses `floor()`, while `index.html` uses continuous (non-floored) growth, and asked whether the HTML was wrong. Checked against all 14 district aggregate rows with a nonzero Assistant Superintendent count: the real values are fractional (e.g. 1.29, 1.37, 1.52, 1.79 positions), not integers, which is only possible with a continuous formula — a floored formula would produce integers only. So `index.html`'s continuous formula was actually correct, and the Rule Register's `floor()` was the error; it's being corrected there instead. (For comparison, Director — which shares the identical LEA-wide base — genuinely is floored, confirmed by exact integer matches; the two positions really are computed differently despite sharing an input.)

The same review raised five more points on 2026-07-24, each checked individually rather than accepted or waved off as a group. (1) It pointed out that PEFC's New Rule Updates tab and the Formula Technical Reference sheet's statute paraphrase both describe the Assistant Superintendent/Director/11-Month Supervisor base as "Division I state units" alone, while this calculator uses a broader combined base — and that an earlier History entry's "match exactly" claim didn't account for this. Tested directly: pure Division I units do *not* reproduce the real workbook's output for these positions (confirmed against every district and charter aggregate row), while the combined base does, exactly or to within ~0.004. So the calculator is faithful to what the workbook actually computes, but that computation conflicts with what PEFC's own documentation says the formula is — a real, unresolved discrepancy, now flagged as an open item in the Register tab and Rule Register rather than glossed over. (2) It asked about a 225-unit boundary claimed in an earlier exchange (149 units → ~0.99, 225 units → 1, not 1.5) that contradicted the current continuous-division implementation for 11-Month Supervisor and Transportation Supervisor. The source `.xlsm` isn't in this project folder, so that earlier claim couldn't be freshly re-verified against real data this session — but the statute text (already in the workbook, on the Formula Technical Reference sheet) is explicit that fractional treatment is only provided for districts below the first threshold, which implies floor above it. The calculator now follows that statutory reading, with status downgraded to "pending" rather than "confirmed" pending access to the source workbook. (3) It correctly noted that Whole-LEA scope in the What-If Model computes one combined hypothetical enrollment rather than summing multiple individual schools, which is not mathematically equivalent given the nonlinear Principal/AP/Admin Support Professionals formulas — now documented explicitly as a real limitation rather than left implicit. (4) It found stale cross-references left over from the 2026-07-23 ID rename: New Rule Updates pointed at pre-rename Rule Register row numbers, and several Rule Register cells said "same as BR-011" where "BR-011" (a valid *old* ID for a since-retired, unrelated row) should have read "BR2-011" — both fixed, six such ID collisions corrected. (5) It listed several already-acknowledged auditability gaps (missing source `.xlsm`, partial Retired Rules reconstruction, pattern-derived Principal/AP/Admin Support Professionals, open Reading Cadre conflict, blank Business Owner/Approver columns) — these were already true and are now stated plainly in Known Limitations rather than scattered across separate notes.

The same review also pointed out that Regular Pre-K was still documented in PEFC's New Rule Updates (row 14) and Weight Register (row 5) even though the calculator doesn't implement it — arguing it should either be added, clearly labeled as a proposed rule, or formally withdrawn by PEFC. Zoey initially asked to add it to the What-If Model with that exact labeling; that work was started (an editable "Regular Pre-K (Proposed PEFC rule — not current law)" input) but then reversed a moment later — Zoey decided PEFC's proposal shouldn't be modeled or documented in this project at all going forward, and asked for it to be removed from the xlsx's New Rule Updates and Weight Register too. Claude pushed back first: those two sheets are meant to be a factual record of what PEFC actually proposed in a real meeting, independent of what this calculator implements, and deleting them would erase that record rather than reflect an actual withdrawal by PEFC (which hasn't happened). Zoey heard the concern and confirmed deletion anyway. The rows were removed from both sheets, with their full original content preserved in a dated note on the Legend sheet (per this workbook's own "never delete without a record" rule) explaining what was removed, why, and that PEFC has not actually withdrawn the proposal — this project simply chose not to carry it in its working documents. The What-If Model's brief re-addition of the Pre-K input was reverted in full (enrollment field, tooltip references, and the Total Enrollment override description), so the calculator, the xlsx, and this README are back in sync: Regular Pre-K is not modeled or documented anywhere in this project as of 2026-07-24.

While fixing New Rule Updates' stale "legal citation pending" status on the five Division I divisor rows (Zoey caught that the Authority/Citation column already had real statute text but the status column hadn't been updated to match — fixed, all five set to Verified), Zoey asked a sharper follow-up: should these five rows even be in New Rule Updates at all, since they're already codified? Checked directly: yes, the concern was right. The "Proposed New Rule" column for all five rows was word-for-word identical to the existing statutory divisor (14 Del. C. § 1703(a)(2)-(6)), and the "districts and charter schools are eligible" scope note wasn't a new PEFC policy either — charter eligibility for these exact categories was already confirmed in real current data and already documented as Verified, District & Charter in the Rule Register (BR2-001 through BR2-005). New Rule Updates exists to track PEFC's *proposed changes*; these five rows proposed no change at all, they just restated settled law, which made it look like open/pending policy when it wasn't. Removed from New Rule Updates, with the full original row content preserved in a dated Legend note. The Rule Register itself — the actual governing document — was untouched, since it already had these five correct.

A final review pass on 2026-07-24 caught a real mistake and sharpened the Locked-vs-What-If distinction that had been getting blurry. First, and most important: the 2026-07-24 floor-above-threshold fix for 11-Month Supervisor and Transportation Supervisor had been applied to *both* the Locked Model and the What-If Model — but the Rule Register's own Boundary Test Cases already listed the real statewide reconciliation targets as 107.77 and 18.84 positions, fractional values a floor formula can never produce. Confirmed directly against the embedded real Statewide row (16,165.934 combined units ÷ 150 = 107.7729; 141,314 enrollment ÷ 7,500 = 18.8419) — the original "continuous, no floor" claim was correct for the Locked Model all along. Reverted the Locked Model back to continuous; the statute-literal floor reading now lives only in the What-If Model, as an explicit alternative to what the real workbook computes rather than a claim about the workbook itself. Second, a related base-of-calculation issue: PEFC's New Rule Updates literally describes the Assistant Superintendent/Director/11-Month Supervisor base as "Division I positions," but the calculator had been using the real workbook's broader combined base (Division I + Principal + AP + Instructional Supports + Admin Support Professionals) in *both* tabs. The Locked Model correctly keeps the combined base, since that's what the real workbook computes. The What-If Model was switched to pure Division I units, since its job is to test PEFC's proposed rule as written — and Assistant Superintendent was also switched from continuous to a floored 300-unit step function there, since PEFC's wording ("1 Position per 300 Division I Positions") reads as a step rule with nothing authorizing fractional positions. The two tabs will now legitimately disagree on hypothetical scenarios — intentional, since it's exactly the PEFC-vs-workbook gap this project exists to surface. Third, several Rule Register fields were self-contradictory and got fixed: BR2-013 (Assistant Superintendent) said "District & Charter" eligibility while its own formula and status said district-only (corrected to District only); BR2-013 and BR2-014 (Director) both said "School (summed to LEA)" scope despite being computed once at the LEA level (corrected to LEA); and the Legend's discrepancy count was stale, still saying Reading Cadre was the only open item when the Rule Register now lists three (Assistant Superintendent, Director, Reading Cadre) — corrected. Finally, language throughout the calculator that described dollar rates as something "typed in by hand" or merely "pre-filled" was corrected — every position and salary rate in the Locked Model is real FY25-26 data drawn directly from PEFC's reference workbook, not a manually-entered placeholder; the only true manual input in that tab is the optional Additional State Funding amount.

A follow-up review caught one real bug in the fixes above: the What-If Model's Whole-LEA scope note always said Assistant Superintendent was 0 because the LEA was below the 300-unit threshold, even in scenarios where the computed value was actually 1 or 2 — fixed to reflect the real computed value. The review also asked for proposed-rule formulas to get the same rule-to-code traceability as the current-workbook Rule Register: added Proposed Rule ID / Formula (Executable) / Boundary Test Cases columns to New Rule Updates, populated for the three components that already have executable proposed-rule logic in the What-If Model (Assistant Superintendent → BR2-PROP-013, Director → BR2-PROP-014, 11-Month Supervisor → BR2-PROP-015), deliberately kept in a separate ID namespace from the Rule Register's BR2- IDs since the proposed and current formulas are genuinely different for these three. The remaining New Rule Updates rows don't have separate proposed-rule IDs yet because PEFC's proposed rule for each already matches the current workbook — there's no second formula to track until that changes. Separately, Zoey caught that the Legend still said Instructional Supports had no defined formula, when the Rule Register has carried a verified one (`div1_units * 0.20`) since earlier this session — corrected; only Administrative Support Professionals genuinely has no independently-derivable formula. The Whole-LEA multi-school rollup (Principal/AP/Admin Support Professionals computed once per LEA rather than summed from individual schools) remains an accurately-disclosed limitation, not yet built.
